<Accordion title="0.4.0 — TanStack Start & Vite React (upcoming)">

**New frontends:** TanStack Start and Vite + React are now first-class citizens. Every auth provider (Clerk, WorkOS, Better Auth) ships a dedicated slice for each new frontend — no combination is blocked.

**Git initialisation:** Every generated project now starts with `git init` + an initial commit. A selection-aware `.gitignore` is generated from your choices — correct entries for your frontend, backend, and auth provider, plus commented `.env.local` key hints for every secret in your stack.

**Bug fixes:** Duplicate `src/` nodes in folder preview, wrong WorkOS redirect URIs, `fe:none` showing `vite` backend in CLI, randomize button ignoring compatibility rules.

[Full release notes →](/docs/changelog/v0-4-0)

</Accordion>

<Accordion title="0.3.0 — WorkOS AuthKit">

**New auth provider:** WorkOS AuthKit support (`--auth workos`) for Next.js projects, using `@workos-inc/authkit-nextjs` middleware with session cookie handling.

[Full release notes →](/docs/changelog/v0-3-0)

</Accordion>

<Accordion title="0.2.0 — Open Source Launch">

**Initial public release** of `create-fusion-stack`. Ships Next.js + Convex, Clerk, Better Auth, Drizzle, Prisma, Mongoose, tRPC, oRPC, shadcn/ui, Resend, PWA, and the visual stack builder.
